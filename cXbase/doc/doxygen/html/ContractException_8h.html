Checks the validity of a predicate f.

This MACRO is usually used as a checkpoint inside a complex procedure to validate the state of the procedure so far. This MACRO should not be used as Pre/postconditions checks.

#define INVARIANT (   f)    if (!(f)) throw InvariantException(__FILE__,__LINE__, #f);

Checks the validity of one predicate f to determine if an invariant (only one) is valid.

This MACRO usually is used in two distinct ways:

  • to check the validity of a unique/specific invariant, i.e. an invariant that cannot be treated in the checkInvariant() method;
  • to implement the checkInvariant() method.
Exceptions
InvariantExceptionIf f is not valid.

#define INVARIANTS ( )    checkInvariant()

Checks the state an object.

The MACRO calls the checkInvariant() method (which has to be defined by the developper for each class). This methods checks the state of various invariants (also defined by the class developper). For example, if you have a People class which has a name member. An invariant for this class could be that the name member should never be empty. Thus, at every time, if you want your object to be valid, you should check that the name member is not empty (i.e. the invariant is respected). This is done using this MACRO.

The nice thing about the checkInvariant() method is that it enables the developper to check many invariants all at once. This is especially easy with this MACRO.
