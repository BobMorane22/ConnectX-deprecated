DesignByContract.h

A classic usage of these design-by-contract exceptions and MACROs can be summarized by these steps:

  • Implement the checkInvariant() methods with the following signature: void checkInvariant() const . In the method body, use the INVARIANT(f) MACRO to test for each class invariant. Each class invariant f must be implemented as a predicate (something that evaluates to true or false). If the class requires the verification of five (5) class invariants, then use the MACRO five times, with the 5 respective predicates. Note that sometimes, a class has no invariant. In this case, do not define the checkInvariant() method.
  • For each method that needs to have precondition(s) verified to ensure a correct behaviour, use the PRECONDITION(f) MACRO in the body. Again, Each class invariant f must be implemented as a predicate. The MACRO must be included in the method body before any other line of code is written for good results. If the method requires the verification of five (5) preconditions, then use the MACRO five times, with the 5 respective predicates.
  • Just like for preconditions, add the required number of postconditions MACROs at the end of the method body. These MACROs must be the last code to be executed before a return statement, or the method's end of scope if no class invariant are defined. Otherwise, they must be the last code to be executed before the INVARIANTS() MACRO (notice the S !).
  • If you have defined class invariant in the first step, add the INVARIANTS() (notice the S !) MACRO before the method's return statement, or before the method goes out of scope for void methods.

Furthermore, if you have complicated methods, in every step of the method's body, you can add the ASSERTION(f) MACRO to test a predicate f. The following example shows how all these tools can be combined.

#include "ContractException.h"
class PositiveValue
{
public:
PositiveValue(int p_value)
{
PRECONDITION(p_value >= 0); // Should be true before anything is done.
m_value = p_value; // For the sake of the example only, no
// initialization list has been used.
if(m_value >= 0)
m_isPositive = true;
POSTCONDITION(m_value >= 0); // Should be true after the operation is done
INVARIANTS(); // Checks both invariants defined in the
// checkInvariant() method below. They should ALWAYS be
// true no matter what, otherwise the object could
// be left in an invalid state.
}
private:
// Needs to be defined by the developper:
void checkInvariant() const
{
INVARIANT(m_value >= 0); // A first, single, invariant.
INVARIANT(m_isPositive = true); // A second, single, invariant.
}
// member variables:
int m_value;
bool m_isPositive;
};
